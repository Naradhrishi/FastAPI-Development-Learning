# ============== MIDDLEWARE NOTES BY AI in easy to understand language =================

# Okay okay, let's restart. Forget the docs language. I'll teach you like you're 10 years into coding, but day 1 into FastAPI.

# Think of middleware as a **security check gate at an airport.**

# ### The Big Idea

# Your API has many gates - `/login`, `/products`, `/home`. 
# Without middleware, a person goes straight to the gate.

# With middleware, EVERY person has to pass through ONE common tunnel before reaching any gate, and again while coming back out.

# That tunnel = Middleware.

# ```
# Request from user
#       |
#       v
#   [ YOUR MIDDLEWARE - ENTRY ]
#       |
#       v
#   Your actual function @app.get("/home") runs
#       |
#       v
#   [ YOUR MIDDLEWARE - EXIT ]
#       |
#       v
# Response to user
# ```

# That's it. That's the whole flow.

# ### Now let's decode the code

# This is the simplest middleware ever:

# ```python
# from fastapi import FastAPI, Request
# import time

# app = FastAPI()

# @app.middleware("http")
# async def my_middleware(request: Request, call_next):
#     print("1. Request came in")
    
#     response = await call_next(request)
    
#     print("2. Response going out")
#     return response
# ```

# Let's break this line by line like a teacher:

# **1. `@app.middleware("http")`**
# You are telling FastAPI: "Hey, this is not a normal API route. This is THE TUNNEL. Run it for every `http` request."

# **2. `async def my_middleware(request: Request, call_next):`**
# FastAPI will automatically give you 2 things when a request comes:

# a) `request` : This is an object containing EVERYTHING about the incoming request.
# - `request.url` -> which url user hit
# - `request.method` -> GET, POST etc
# - `request.headers` -> browser info
# - `request.client.host` -> IP of user

# It's just a box of information.

# b) `call_next` : This is NOT the response. This is a **function**. A magic button.
# When you press this button, FastAPI says "Okay, I will now go and run the actual API function that user wanted, like `/home`".

# If you never press `call_next`, the request will never reach your API. It will be stuck in the tunnel.

# **3. `response = await call_next(request)`**
# This is the most important line. Two things happen here:

# - `call_next(request)` = "Go run the real API function now"
# - `await` = "Wait until that API function finishes its work, I'm not going anywhere"

# So your code PAUSES at this line. FastAPI goes to your `@app.get("/")` function, runs it, gets the result.

# When it comes back, whatever your API returned is now stored in `response`.

# **4. Before and After**

# This is why we write code above and below that line:

# ```python
# # CODE HERE runs BEFORE your API
# # Good for: checking login, logging, blocking

# response = await call_next(request)

# # CODE HERE runs AFTER your API
# # Good for: adding headers, logging time, modifying response
# ```

# Real example:

# ```python
# @app.middleware("http")
# async def add_time_middleware(request: Request, call_next):
#     # BEFORE
#     start = time.time()
#     print(f"Someone asked for {request.url}")

#     # GO TO REAL API
#     response = await call_next(request)

#     # AFTER
#     total_time = time.time() - start
#     print(f"It took {total_time} seconds")
    
#     # You can even change the response
#     response.headers["X-My-Custom-Header"] = "Hello from middleware"
    
#     return response
# ```

# What happens when user hits `/products`?

# 1.  Print: "Someone asked for /products"
# 2.  Start timer
# 3.  `await call_next(request)` -> goes to your `def get_products():` function
# 4.  `get_products()` runs and returns `{"products": [...]}` 
# 5.  Comes back, calculates time
# 6.  Print: "It took 0.02 seconds"
# 7.  Adds header to response
# 8.  Returns to user

# ### Under the hood - what you should actually imagine

# FastAPI does NOT run your middleware after your route. It WRAPS your route inside your middleware.

# It's like this in Python pseudo-code:

# ```python
# def middleware(request):
#     # your before code
#     response = your_real_api_function(request) # this is call_next
#     # your after code
#     return response
# ```

# So `call_next` is just FastAPI's way of saying "the inner function".

# ### One rule you must never forget

# **You MUST return a response.**

# If you do:
# ```python
# response = await call_next(request)
# # forgot to return
# ```
# User will get nothing. Always `return response`.

# And if you want to block a request, just DON'T call `call_next`:

# ```python
# @app.middleware("http")
# async def block_middleware(request: Request, call_next):
#     if request.client.host == "bad guy IP":
#         from fastapi.responses import JSONResponse
#         return JSONResponse({"error": "Blocked"}, status_code=403)
    
#     # otherwise allow
#     return await call_next(request)
# ```

# See? If you block, you return your own response. If you allow, you call `call_next`.
